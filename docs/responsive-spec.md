# 职业墓场响应式设计基准

## 设计基准
- `1440 x 1080`：标准桌面主稿，作为视觉基准。
- `1280 x 960`：窄桌面主稿，优先保证导航与标题可读性。
- `768 x 1024`：平板基准，布局开始从展示型转向阅读型。
- `390 x 844`：手机基准，保留首页与归档的两列墓碑阵列。

## 容器与边距
- 双容器策略固定：
  - `content-narrow = 1280px`
  - `content-wide = 1440px`
- 页面内容不得无限拉宽，主内容区域必须受上述双容器约束。
- 页面左右 gutter：
  - `390`：`16px`
  - `768`：`24px`
  - `1280`：`32px`
  - `1440`：`32px`

## 导航规则
- 全站导航统一为白色底色的极简样式，不再依赖 `mix-blend-mode`。
- 导航默认使用接近纯白的实底，以避免与页面正文和英雄区文字冲突。
- 导航保留底部细分隔线，作为全站固定层与正文之间的边界。
- 导航宽度使用全宽布局，不再锁进内容容器。
- Logo 使用黑色前景，菜单默认使用 `text-neutral` 风格，激活项切为黑色。
- 返回链接始终使用黑色，hover 只降低透明度。
- 导航微字体基准为 `0.725rem`，移动端不低于 `0.7rem`。

## 字号与点击规则
- 首页主标题使用 `clamp()`：
  - `390`：约 `2.35rem`
  - `768`：约 `2.9rem - 4.2rem`
  - `1280`：约 `3.2rem - 4.35rem`
  - `1440`：约 `3.35rem - 4.7rem`
- 页面标题使用 `clamp()`：
  - `390`：约 `2.25rem`
  - `768`：约 `2.7rem - 4.2rem`
  - `1280`：约 `3rem - 4.4rem`
  - `1440`：约 `3.1rem - 4.8rem`
- 主要按钮文字不低于 `0.75rem`，最小点击高度保持 `48px`。

## 栅格规则
- 首页墓碑阵列：
  - `390`：2 列
  - `768`：3 列
  - `1280`：6 列
  - `1440`：6 列
- 归档页墓碑阵列：
  - `390`：2 列
  - `768`：4 列
  - `1280`：6 列
  - `1440`：8 列
- 相关职业阵列：
  - `390`：2 列
  - `768+`：4 列

## 页面专属布局
- 首页：
  - 首屏标题、问题文案、竖线和墓碑阵列需要保持稳定的纵向节奏。
  - `390` 下仍保持两列阵列，不切单列。
- 归档：
  - `390` 下保持两列紧凑阵列。
  - 筛选区在 `768` 以下切为上下结构。
- 详情：
  - `390` 必须为单列阅读。
  - `768` 允许弱双列，但阅读顺序优先。
  - `1280 / 1440` 使用双列信息布局。
- 祭奠：
  - `390 / 768` 下表单与内容区使用单列流式布局。
- 关于：
  - 方法论、统计区、贡献者网格在 `390` 下必须降密度展示。

## 验收规则
- Playwright 必须覆盖四套正式项目：
  - `desktop-1440`
  - `desktop-1280`
  - `tablet-768`
  - `mobile-390`
- 必须保留稳定截图基线：
  - 首页整页
  - 首页导航局部
  - 归档整页
  - 详情整页
- 验收重点：
  - `1440 / 1280` 下导航必须一眼可见且可点击。
  - `768` 下页面层级不能崩。
  - `390` 下首页与归档继续保留两列阵列感。
