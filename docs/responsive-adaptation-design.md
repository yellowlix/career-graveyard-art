# Career Graveyard 响应式适配设计文档

## 1. 文档目的

本文档用于固化当前站点的响应式适配方案，作为设计评审、前端实现和回归测试的统一依据。当前版本采用四档宽度基线：

- `390`：手机基线
- `768`：平板基线
- `1280`：桌面基线
- `1440`：宽桌面基线

文档覆盖页面范围：

- `home`
- `archive`
- `detail`
- `memorial`
- `about`
- `404`

## 2. 断点与容器基线

### 2.1 宽度基线

- `390 x 844`
  - 对应 Playwright 项目：`mobile-390`
  - 目标：手机阅读优先，保留首页和归档的双列墓碑感
- `768 x 1024`
  - 对应 Playwright 项目：`tablet-768`
  - 目标：从展陈式布局转向阅读式布局，主要内容开始纵向堆叠
- `1280 x 960`
  - 对应 Playwright 项目：`desktop-1280`
  - 目标：标准桌面阅读密度，导航稳定，多栏布局恢复
- `1440 x 1080`
  - 对应 Playwright 项目：`desktop-1440`
  - 目标：宽桌面展示基线，区块留白更舒展，归档列数最高

### 2.2 容器策略

- `content-narrow = 1280px`
- `content-wide = 1440px`

页面不采用无限拉宽的流体布局，而是在不同断点下通过固定最大内容宽度控制阅读密度：

- 主要正文和常规页面内容受 `content-narrow` 约束
- 首页首屏、归档阵列等更宽的展示区受 `content-wide` 约束

### 2.3 页面边距

- `390`：`16px`
- `768`：`24px`
- `1280`：`32px`
- `1440`：`32px`

## 3. 全局响应式规则

### 3.1 导航

- 全站导航在四档宽度下都必须保持可见、可点击
- 首页使用更轻的 `site-nav--home` 视觉变体，内页使用常规 `site-nav--page`
- `<= 767` 时，导航允许换行和纵向堆叠
- `<= 767` 时，语言切换器缩小尺寸，但仍需保持清晰点击区域
- 导航文案遵循单语切换规则：
  - 中文态为纯中文
  - 英文态为纯英文

### 3.2 页脚

- 页脚在桌面端保持横向信息分布
- `<= 1279` 时，页脚内层转为单列堆叠
- 四档宽度下页脚链接都必须可见且可点击

### 3.3 标题与层级

- 首页主标题使用 `hero-title-size`
- 其他页面主标题使用 `page-title-size`
- 标题尺寸随断点提升，但层级关系不变：
  - 主标题始终最大
  - 副标题、眉题、状态标签保持更轻、更疏的节奏

### 3.4 布局切换总原则

- `1280+`：恢复多栏桌面布局
- `<= 1279`：优先阅读流，关键区块进入纵向堆叠
- 手机和平板不是“桌面缩小版”，而是重新组织阅读顺序

## 4. 页面级适配规则

### 4.1 首页 `home`

目标：海报式首屏 + 中轴提问 + 墓碑阵列

#### 栅格

- `390`：`2` 列
- `768`：`3` 列
- `1280`：`6` 列
- `1440`：`6` 列

#### 规则

- 手机下首页仍保持双列墓碑阵列，不退化为单列
- 平板下首屏留白收紧，但中轴提问关系保留
- 桌面下首屏居中构图完整呈现，导航悬浮感更强
- 引用区和 CTA 在窄屏下转为更紧凑的纵向阅读节奏

### 4.2 归档页 `archive`

目标：清晰的归档检索体验和高密度墓碑列表

#### 栅格

- `390`：`2` 列
- `768`：`4` 列
- `1280`：`6` 列
- `1440`：`8` 列

#### 规则

- 手机下归档仍需保留双列感
- `<= 1279` 时，筛选和排序控制区转为单列堆叠
- `1440` 下归档密度最高，列数必须高于窄屏

### 4.3 详情页 `detail`

目标：阅读优先的墓志铭页面

#### 规则

- `390`：单列阅读
- `768`：单列阅读，不维持桌面式并排信息布局
- `1280 / 1440`：恢复桌面信息区与正文区的多栏关系
- 相关推荐区列数：
  - `390`：`2` 列
  - `768+`：`4` 列

### 4.4 祭奠页 `memorial`

目标：邮箱投稿 + 静态示例说明

#### 规则

- `390 / 768`：表单区与示例区必须纵向堆叠
- `1280 / 1440`：表单区与示例区为双列布局
- 静态示例说明必须可见，不能因为窄屏折叠而消失
- 邮件草稿预览与回退说明在四档宽度下都要可读

### 4.5 关于页 `about`

目标：静态站点说明、方法论、标准与贡献者信息

#### 规则

- `390 / 768`：双栏信息区改为单列阅读
- `1280 / 1440`：恢复双栏结构
- 方法论、信息卡片、统计带在 `<= 1279` 时统一转为单列
- 贡献者区在手机下允许降为更紧凑的双列，但不可溢出

### 4.6 404 页面

目标：在四档宽度下都保持明确、克制、易返回

#### 规则

- 错误标题、说明和返回动作始终在首屏主要区域内可读
- 操作按钮在窄屏下允许纵向排列
- 页面不能出现横向滚动

## 5. 共享组件响应式行为

### 5.1 NavigationBar

- 四档宽度下均保持可见
- 首页与内页可使用不同视觉变体，但交互逻辑一致
- 手机下允许：
  - logo 单独占一行
  - 导航链接换行
  - 语言切换器换到下一行

### 5.2 PageFooter

- 桌面下为横向排布
- `<= 1279` 下转为单列堆叠
- 首页页脚可以更轻，但链接信息结构不得丢失

## 6. 测试矩阵

### 6.1 截图基线页

以下页面继续维持四档截图基线：

- `home`
- `archive`
- `detail`

截图命名沿用现有规则：

- `*-desktop-1440`
- `*-desktop-1280`
- `*-tablet-768`
- `*-mobile-390`

### 6.2 结构断言页

以下页面采用四档结构与可读性断言：

- `memorial`
- `about`
- `404`

断言重点：

- 主标题可见
- 关键内容区可见
- 在平板与手机下进入堆叠态
- 在桌面下恢复多栏或更高密度
- 页面无明显横向溢出
- 导航与页脚在四档下均可见

## 7. 验收标准

### 7.1 视觉验收

- `home / archive / detail` 四档截图通过
- 截图需保持当前视觉节奏，不允许因断点回归导致结构错位

### 7.2 结构验收

- `memorial / about / 404` 四档断言通过
- 至少验证以下场景：
  - 手机下首页和归档保持双列感
  - 平板下详情、祭奠、关于进入纵向阅读布局
  - 桌面下归档列数高于窄屏
  - 导航与页脚在四档下可见且可点击

### 7.3 通用约束

- 页面不得出现明显横向滚动
- 关键文字区不可被遮挡
- 主要交互元素不得落入不可点击状态
- 所有页面仍遵循当前单语切换逻辑，不回退到中英混排

## 8. 当前实施结论

当前站点的响应式方案并非“完全流体自适应”，而是围绕 `390 / 768 / 1280 / 1440` 四个基线进行控制。设计评审与回归测试都应优先以这四档为准，新增页面或大改布局时，也应先补齐这四档下的表现与测试覆盖。
